合併排序法 Merge Sort

合併排序採用分治法：先把陣列持續切成更小片段，等到每段都足夠小後，再把兩段已排序資料線性合併回來。

核心原理

切分到單一元素後，再透過 merge 將兩段有序資料合併成一段更大的有序區間。

觀察重點

時間複雜度穩定為 O(n log n)，但需要額外陣列空間來進行合併。

流程拆解

操作過程的步驟

  • 1. 先切分區間 每次把區間切成左右兩半，直到每段只剩一個元素。
  • 2. 左右各自排序完成 透過遞迴，先處理左半，再處理右半，確保子問題都已排序。
  • 3. 線性合併兩段有序陣列 以雙指標比較左右頭部元素，將較小者依序寫回原陣列。
  • 4. 回到最外層得到完整排序 最後一次 merge 完成後，整個陣列即為遞增排序。
重點觀察：Merge Sort 比較次數很多，但每次合併都維持線性，整體是 O(n log n)。
複雜度

時間與空間成本

情況 時間複雜度 說明
最佳情況 O(n log n) 切分與合併結構固定，不受初始排列影響太多。
平均情況 O(n log n) 每層合併總成本為 O(n)，共約 log n 層。
最差情況 O(n log n) 即使資料逆序，複雜度仍保持在 O(n log n)。
空間複雜度 O(n) 需要暫存陣列輔助 merge，不是原地排序。
演算法觀念

簡化版流程

mergeSort(left, right):
    if left >= right: return
    mid = (left + right) // 2
    mergeSort(left, mid)
    mergeSort(mid + 1, right)
    merge(left, mid, right)
合併排序適合大資料量，且在需要穩定排序時常是實務首選之一。
