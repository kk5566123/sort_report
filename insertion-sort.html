插入排序法 Insertion Sort

插入排序會從左到右逐一取出元素，並插入到左側已排序區間中的正確位置。它的核心是「左側保持有序，右側逐步插入」。

核心原理

每輪取 key，向左比較並搬移較大值，最後把 key 放到空出的位置。

觀察重點

資料越接近有序，插入排序越快；反序時搬移次數會明顯增加。

目前執行程式碼
for (i = 1; i < n; i++) {
    key = arr[i];
    j = i - 1;
    while (j >= 0 && arr[j] > key) {
        arr[j + 1] = arr[j];
        j--;
    }
    arr[j + 1] = key;
}
流程拆解

操作過程的步驟

  • 1. 取出目前元素 key 從索引 i 開始，把當前值暫存成 key，左側視為已排序區。
  • 2. 向左比較並搬移 只要左邊元素比 key 大，就往右搬一格，替 key 騰出空間。
  • 3. 找到位置後插入 key 當不再需要搬移時，把 key 填入 j + 1 的位置。
  • 4. 已排序區逐步擴大 每輪結束後，左側有序區長度加一，直到整列完成。
重點觀察：插入排序不是大量交換，而是連續搬移後再插入。
複雜度

時間與空間成本

情況 時間複雜度 說明
最佳情況 O(n) 資料幾乎有序時，每輪只需少量比較，幾乎不用搬移。
平均情況 O(n^2) 通常需要多次向左比較與搬移，成本與逆序程度有關。
最差情況 O(n^2) 若完全反序，每輪都需要把元素搬到最前端。
空間複雜度 O(1) 只使用 key 與索引變數，屬於原地排序。

簡化版流程

for 每一輪 i (從 1 開始):
    key = arr[i]
    j = i - 1
    while j >= 0 且 arr[j] > key:
        arr[j + 1] = arr[j]
        j -= 1
    arr[j + 1] = key
插入排序在小型資料或近乎排序完成的資料上，常比一般 $O(n^2)$ 演算法更實用。
